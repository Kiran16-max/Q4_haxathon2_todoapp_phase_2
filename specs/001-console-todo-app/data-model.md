# Data Model: Console Todo App (In-Memory)

## Task Entity

### Attributes
- **id**: Integer (unique, auto-incremented)
  - Purpose: Unique identifier for each task
  - Constraints: Positive integer, auto-generated
- **title**: String (required, non-empty)
  - Purpose: The main description of the task
  - Constraints: Required field, non-empty, max length 200 characters
- **description**: String (optional)
  - Purpose: Additional details about the task
  - Constraints: Optional field, max length 1000 characters
- **completed**: Boolean (default: False)
  - Purpose: Track whether the task is completed
  - Constraints: Boolean value, defaults to False

### Validation Rules
1. **Title Required**: Task.title must not be empty or null
2. **Title Length**: Task.title must be between 1 and 200 characters
3. **Description Length**: Task.description, if provided, must be between 1 and 1000 characters
4. **ID Uniqueness**: Each Task.id must be unique within the system
5. **ID Auto-generation**: Task.id values are automatically generated and incremented

### State Transitions
- **New Task**: When created, Task.completed = False by default
- **Mark Complete**: Task.completed transitions from False to True
- **Mark Incomplete**: Task.completed transitions from True to False

## In-Memory Storage Structure

### Task Storage
- **Data Structure**: Dictionary with ID as key, Task object as value
- **Example**: `{1: Task(id=1, title="Buy groceries", description="Milk, bread, eggs", completed=False)}`
- **Auto-increment**: New task IDs are generated by incrementing the highest existing ID

### Storage Operations
- **Create**: Add new task to dictionary with next available ID
- **Read**: Retrieve task by ID from dictionary
- **Update**: Modify task properties in dictionary
- **Delete**: Remove task from dictionary by ID
